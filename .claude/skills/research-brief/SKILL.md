# research-brief — 調べて要点にまとめる

テーマを受け取り、**出典つきの要点ブリーフ**を作ります。長いレポートではなく、判断に使える短いまとめを目指します。

## 手順
1. **問いを絞る** — 何を知りたいのかを一文にする（曖昧なら1つだけ確認質問）
2. **複数ソースを当たる** — 1つの情報源で結論を出さない。最低3つ突き合わせる
3. **事実と意見を分ける** — 「確認できた事実」「推測」「未確認」をはっきり区別する
4. **要点に圧縮** — 箇条書きで、判断に効く順に並べる
5. **出典を添える** — 各要点にソースのリンク/名前を残す

## 出力フォーマット
```
# ブリーフ: <テーマ>

## 結論（30秒で読める）
- <一番大事な要点>
- ...

## 根拠
- <要点> — 出典: <リンク/名前>
- ...

## 注意・未確認
- <鵜呑みにできない点・確認できなかった点>

## 出典一覧
- <リンク>
```

## ルール
- **わからないことを「たぶん」で埋めない。** 未確認は未確認と書く
- 数字・固有名詞は出典とセットで
- 古い情報に注意（いつ時点の話かを確認する）
- 1次情報（公式・統計）を、2次情報（まとめ記事）より優先する
